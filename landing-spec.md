# ТЗ: Лендинг сервиса диагностики сайтов

## Контекст проекта

Создаём лендинг для AI-сервиса, который диагностирует сайты малого бизнеса и показывает, почему они теряют клиентов. Пользователь вводит URL → получает бесплатный экспресс-диагноз → может купить полный отчёт за 1490 ₽ или заказать переделку сайта.

**Это первая страница продукта.** Никакой бэкенд-логики на этом этапе не нужно — только визуал, тексты, форма ввода URL и заглушка отправки.

---

## Технический стек

- **Next.js 14** (App Router)
- **TypeScript** (strict mode)
- **Tailwind CSS**
- **shadcn/ui** для базовых компонентов (Button, Input, Card, Accordion)
- **Framer Motion** для анимаций
- **Lucide React** для иконок
- **Деплой:** Vercel

## Визуальное направление

**Стиль:** чистая медицинская эстетика с одним характерным персонажем-роботом. Светлая тема, тёплые акценты, без кислотных цветов и без классического больничного синего.

### Цветовая палитра

```css
--bg-primary: #FAFAF7;          /* тёплый off-white фон */
--bg-secondary: #FFFFFF;        /* карточки */
--bg-tertiary: #F4F2ED;         /* приглушённые блоки */

--text-primary: #1A1A1A;        /* основной текст */
--text-secondary: #5C5C5C;      /* вторичный */
--text-muted: #9A9A9A;          /* подписи */

--accent-primary: #E53E3E;      /* медицинский красный — для CTA, акцентов */
--accent-secondary: #4FD1C5;    /* мятный — для иллюстраций, иконок */
--accent-warm: #E8B14B;         /* амбер — связка с Zobnin AI, тонко */

--border: #E8E5DE;              /* границы */
--success: #38A169;             /* "здоровые" метрики */
--warning: #DD6B20;             /* "проблемные" метрики */
--danger: #E53E3E;              /* критичные проблемы */
```

### Типографика

- **Заголовки:** `Unbounded` (Google Fonts), weight 500–600, tracking-tight
- **Текст:** `Inter` (Google Fonts), weight 400–500
- **Моноширинный (для URL, технических деталей):** `JetBrains Mono`

Размеры:
- H1: 48–64px на десктопе, 32–40px на мобильном
- H2: 36–48px / 28–32px
- H3: 24–28px / 20–24px
- Body: 17px / 16px
- Small: 14px

### Общие принципы

- Скруглённые углы: `rounded-2xl` для карточек, `rounded-xl` для кнопок
- Тени мягкие, не резкие: `shadow-[0_4px_24px_rgba(0,0,0,0.04)]`
- Spacing щедрый: `py-24` между секциями на десктопе, `py-16` на мобильном
- Анимации тонкие: fade-in + slight-up при скролле, без bouncy-эффектов
- Никаких градиентов на фоне всей секции — только акцентные пятна

### Робот-персонаж

Центральная метафора. Вариантов реализации:

1. **Lottie-анимация** (предпочтительно) — найти/заказать готовую анимацию робота-доктора. На этом этапе можно использовать плейсхолдер из LottieFiles.
2. **SVG-иллюстрация** — статичная, но с анимированными элементами (моргает, мигают индикаторы)
3. **3D-рендер** — отдельным изображением

Для MVP лендинга: **SVG-плейсхолдер** в `public/robot-placeholder.svg` с пометкой `// TODO: заменить на финальную иллюстрацию`. Закладываем место в layout, но не блокируем разработку отсутствием финального арта.

### Правая колонка

Робот-иллюстрация. Размер примерно 480×480px на десктопе, 320×320px на мобильном.

Под иллюстрацией мелким текстом (text-muted):
> Сегодня уже осмотрено: **[N] сайтов**

Где `[N]` — статичное число для MVP (например, 1247). Позже будет подтягиваться из БД.

### Поведение формы

При отправке:
1. Валидация URL на клиенте (простая regex или `new URL()` в try/catch, добавлять `https://` если не указан)
2. Если невалидный — красная подсказка под полем "Проверьте адрес сайта"
3. Если валидный — пока что просто `console.log(url)` и редирект на `/scanning?url=...` (страница `/scanning` не делается в этом ТЗ, можно сделать заглушку)

```tsx
// pseudo
function handleSubmit(url: string) {
  const normalized = normalizeUrl(url); // добавить https://, убрать пробелы
  if (!isValidUrl(normalized)) {
    setError("Проверьте адрес сайта");
    return;
  }
  router.push(`/scanning?url=${encodeURIComponent(normalized)}`);
}
```

## Блок [6] FAQ

**Заголовок:**
> Частые вопросы

**Аккордеон** на основе shadcn/ui Accordion. Все вопросы свёрнуты по умолчанию.

### Вопросы

**1. А вы не украдёте мой сайт или мои данные?**

Нет. Мы открываем сайт так же, как любой посетитель из браузера — никакого доступа изнутри нам не нужно. Анализируем только то, что видно публично каждому, кто зашёл по ссылке.

**2. Чем это отличается от обычного SEO-аудита?**

SEO-аудит проверяет, видит ли вас Google. Мы проверяем другое — понимает ли вас клиент, который уже зашёл на сайт. Это разные вещи, и обычно проблема именно во втором: люди приходят, но не покупают.

**3. Что входит в платный отчёт за 1490 ₽?**

Полный разбор главной страницы, готовые тексты для главного экрана и кнопок, рекомендации по структуре, SEO-заголовок, мета-описание. Всё в одном PDF, который можно отдать дизайнеру или сделать самому.

**4. А вы можете переделать сайт целиком?**

Да. Если после диагноза вы поймёте, что проще не чинить, а переделать — напишите, обсудим. Это уже отдельная услуга, делаем в студии {BRAND.parentBrand}.

**5. Какие сайты вы можете проверить?**

Любые публичные сайты на любом конструкторе: Tilda, WordPress, Bitrix, Webflow, самописные. Главное — чтобы сайт был доступен по ссылке.

**6. Сколько ждать результата?**

Меньше минуты. Бесплатный диагноз — сразу после сканирования. Полный отчёт открывается мгновенно после оплаты — он готовится одновременно с бесплатным.

**7. Что если мой сайт получит низкую оценку?**

Это нормально и даже полезно. Низкая оценка — это карта действий: видно, что чинить и в каком порядке. Большинство сайтов малого бизнеса получают 30–55 баллов из 100, и это не катастрофа, а отправная точка.

## Анимации

Используем Framer Motion. Принципы:

1. **Появление при скролле** — `whileInView` с `opacity: 0 → 1`, `y: 20 → 0`, длительность 0.5s, easeOut
2. **Stagger для карточек** — задержка 0.1s между соседними элементами
3. **Hover на карточках** — `whileHover={{ y: -4 }}`, transition spring
4. **Робот в hero** — лёгкая идле-анимация (если SVG): покачивание ±2deg, 4s loop
5. **Кнопки** — scale 1 → 0.98 при tap

**НЕ использовать:** parallax, бесконечные карусели, авто-играющие видео, тяжёлые WebGL.

## Финальное напоминание

Главное в этом лендинге — **доверие к диагностике**. Пользователь должен поверить, что AI-доктор реально что-то понимает в сайтах, а не выдаёт банальности. Это создаётся:

1. Тоном "доктора" в текстах (прямой, конкретный, без воды)
2. Качеством блока «Пример отчёта» — он должен выглядеть как настоящий разбор
3. Минимализмом интерфейса — никаких маркетинговых трюков и стоковых улыбок

Если в коде где-то возникает соблазн добавить «Доверяют 500+ компаний» с фейковыми логотипами — не добавляй. Лучше пустота, чем фальшь.
