### Логотип

Логотип — текстовый wordmark + иконка слева.

**Иконка:** медицинский крест в круге, но стилизованный под "tech-крест":
- Круг с тонкой обводкой
- Внутри — крест, у которого горизонтальная перекладина имеет вид графика/пульса (зигзаг как на ЭКГ)
- Цвет иконки на светлом фоне: `#0A0E1A` (почти чёрный)
- Цвет иконки на тёмном фоне: `#00D4FF` (cyan)

**Wordmark:** "Site Doctor", шрифт Unbounded weight 600, tracking-tight.
Между Site и Doctor — обычный пробел, не схлопывать.

**Расположение:** иконка слева, текст справа, gap-2.

Файлы создать в `public/logo/`:
- `logo-light.svg` — для светлых блоков
- `logo-dark.svg` — для тёмных блоков
- `icon.svg` — только иконка (для favicon)

Если делать иконку быстро — использовать Lucide `HeartPulse` внутри круга как временное решение, пометить `// TODO: заменить на финальный wordmark`.

## 2. Гибридная визуальная стратегия

### Принцип

```
[Светлый лендинг]
   ↓ нажал "Проверить сайт"
[Тёмная страница сканирования — лаборатория]
   ↓ сканирование завершено
[Тёмный отчёт-превью с диагнозом]
   ↓ хочет полный отчёт
[Светлый чекаут / форма оплаты]
   ↓ оплачено
[Тёмный полный отчёт]
   ↓ хочет переделку сайта
[Светлый лендинг done-for-you / контакт]
```

**Логика:**
- Светлая зона = решения, доверие, действие пользователя
- Тёмная зона = диагностика, экспертная работа, "лаборатория"

Контраст создаёт театральный эффект: пользователь "входит в кабинет врача". Это и есть продукт.

### Палитра для тёмной зоны

Добавить в `globals.css` рядом со светлыми переменными:

```css
:root {
  /* ... светлая палитра остаётся как было ... */
}

[data-theme="dark"], .dark-zone {
  /* Фоны */
  --bg-primary: #0A0E1A;          /* почти чёрный с синим подтоном */
  --bg-secondary: #111827;        /* панели */
  --bg-tertiary: #1A2237;         /* приподнятые элементы */
  --bg-elevated: #232D45;         /* hover, focus */

  /* Текст */
  --text-primary: #E8ECF4;
  --text-secondary: #9BA8C0;
  --text-muted: #5A6783;

  /* Границы */
  --border: #1F2A44;
  --border-glow: rgba(0, 212, 255, 0.3);

  /* Акценты — cyan + красный из референсов */
  --accent-primary: #00D4FF;       /* основной HUD-цвет */
  --accent-secondary: #FF3B5C;     /* критичные проблемы, низкий score */
  --accent-warm: #FFB547;          /* warning, предупреждения */
  --accent-success: #00E5A0;       /* положительные метрики */

  /* Эффекты */
  --glow-cyan: 0 0 24px rgba(0, 212, 255, 0.4);
  --glow-red: 0 0 24px rgba(255, 59, 92, 0.4);
}
```

### Применение классов

Тёмная зона применяется через класс `.dark-zone` на корневом элементе страницы:

```tsx
// app/scanning/page.tsx
<main className="dark-zone min-h-screen bg-[var(--bg-primary)] text-[var(--text-primary)]">
  ...
</main>
```

Не используем `next-themes` или системную тёмную тему — это **зональное решение, не пользовательская настройка**.

### Шрифты в тёмной зоне

В тёмной зоне добавляется моноширинный акцент:
- Числа, score, технические данные — `JetBrains Mono` weight 500
- Заголовки и обычный текст — те же Unbounded / Inter

## 3. Изменения в лендинге

Лендинг **остаётся светлым**, как описано в `landing-spec.md`. Но добавляются три модификации, которые намекают на стиль внутренней зоны.

### 3.1. Робот в Hero — заменить SVG-плейсхолдер на референсное изображение

В `landing-spec.md` было: «SVG-плейсхолдер робота».

**Меняем на:** статичное изображение робота-доктора в стиле референсов.

Файл: `public/robot/hero-robot.png` (1024×1024px, прозрачный фон или светлый).

**Подготовка ассета (вне Cursor — это твоя задача до запуска):**
- Использовать одно из загруженных референсных изображений робота
- Вырезать робота с фоном через любой background remover (remove.bg, photoshop)
- Сохранить как PNG с прозрачностью
- Также подготовить `hero-robot@2x.png` для retina

В коде:
```tsx
import Image from "next/image";

<Image
  src="/robot/hero-robot.png"
  alt="Site Doctor — AI-диагност сайтов"
  width={520}
  height={520}
  priority
  className="drop-shadow-2xl"
/>
```

**Важно:** на светлом фоне лендинга у робота должна быть мягкая тень снизу (`drop-shadow-2xl`) и опционально лёгкое свечение cyan через CSS:

```css
.robot-glow {
  filter: drop-shadow(0 0 60px rgba(0, 212, 255, 0.15));
}
```

Это создаёт тонкую визуальную связь с тёмной зоной, но не делает лендинг тёмным.

В оригинальном ТЗ блок "Пример отчёта" был светлой карточкой. **Меняем на тёмную диагностическую панель**, встроенную в светлую страницу. Контраст создаёт wow-момент.

- `public/sample/preview-fix.png` — мини-макет "после". Можно нарисовать в Figma на скорую руку или взять референс.

### 3.3. Тонкие cyan-акценты на светлом лендинге

Добавить в светлую палитру дополнительный акцент:


Применять **очень дозированно**:
- Подчёркивание под ключевыми словами в заголовках секций

- Hover-состояния на ссылках "хотите такой по своему сайту"

Это намекает на внутреннюю эстетику, но не делает лендинг "техно".

## 5. Страница `/scanning` — расширенная заглушка

В оригинальном ТЗ `/scanning` была "пустой заглушкой". **Делаем её визуально значимой**, потому что это первая тёмная страница, на которую попадает пользователь.

### Логика

После отправки формы с лендинга → редирект на `/scanning?url=...`

В MVP **никакого реального сканирования нет**. Страница просто:
- Принимает URL из query
- Показывает анимацию "сканирования" 8–12 секунд
- Через таймаут редиректит на `/report/sample` (статичная демо-страница)

### Дизайн

Полностью тёмная зона (`<main className="dark-zone">`).

```
[По центру экрана]

  [Робот-доктор — изображение scanning-robot.png, 320×320]
  
  WEBSITE X-RAY
  Анализируем sitename.ru...
  
  [Прогресс-список, чекбоксы появляются по очереди]
  ✓ Открываем главную страницу
  ✓ Сканируем мобильную версию
  ⏳ Анализируем тексты и призывы
  ○ Проверяем сигналы доверия
  ○ Формируем диагноз
  
  [Тонкий прогресс-бар, cyan, с glow]
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 64%
  
  [Микротекст внизу]
  Обычно занимает 30-60 секунд. Не закрывайте страницу.
```

### Реализация

```tsx
"use client";

import { useEffect, useState } from "react";
import { useSearchParams, useRouter } from "next/navigation";

const STEPS = [
  { id: 1, label: "Открываем главную страницу", duration: 1500 },
  { id: 2, label: "Сканируем мобильную версию", duration: 2000 },
  { id: 3, label: "Анализируем тексты и призывы", duration: 2500 },
  { id: 4, label: "Проверяем сигналы доверия", duration: 2000 },
  { id: 5, label: "Формируем диагноз", duration: 2000 }
];

export default function ScanningPage() {
  const params = useSearchParams();
  const router = useRouter();
  const url = params.get("url") || "";
  const [currentStep, setCurrentStep] = useState(0);

  useEffect(() => {
    let elapsed = 0;
    const timeouts: NodeJS.Timeout[] = [];
    
    STEPS.forEach((step, idx) => {
      elapsed += step.duration;
      timeouts.push(
        setTimeout(() => setCurrentStep(idx + 1), elapsed)
      );
    });
    
    timeouts.push(
      setTimeout(() => router.push("/report/sample"), elapsed + 500)
    );
    
    return () => timeouts.forEach(clearTimeout);
  }, [router]);

  return (
    <main className="dark-zone min-h-screen flex items-center justify-center p-6">
      {/* layout как описано выше */}
    </main>
  );
}
```

**Важно:** для MVP это полностью клиентская иллюзия. Когда подключим реальный воркер — UI остаётся тот же, меняется только источник прогресса (Supabase Realtime вместо setTimeout).

### Контент демо-отчёта

Это статичный контент, прописанный в коде. Не использовать никакой динамики на этом этапе.

**Пациент:** условная стоматология
**Скор:** 42/100
**Диагноз (6 пунктов):**
- ✗ Clear Value Proposition — Critical
- ✗ Strong CTA — Critical
- ⚠ Mobile Optimization — Warning
- ⚠ Page Speed — Warning
- ✗ Trust Signals — Critical
- ✓ SEO Structure — OK

**Top Issues (3 карточки):**

1. **Непонятно, чем вы занимаетесь**
   Зашёл на главную и за первые 8 секунд не понял: вы лечите детей или взрослых, делаете имплантацию или только чистку, в Москве вы или в Подмосковье. Это первая причина, по которой клиенты закрывают вкладку.
   `Критично`

2. **Кнопка не объясняет, что произойдёт**
   На главной кнопка "Подробнее" — это самый слабый призыв из возможных. Клиент не понимает: после клика он попадёт в каталог услуг, заполнит форму, или его наберёт менеджер. Когда непонятно — не нажимают.
   `Критично`

3. **Нет ни одного отзыва, кейса или фото врачей**
   В стоматологии доверие строится на лицах. Если на главной нет фотографий врачей, нет отзывов реальных пациентов и нет фото "до/после" — клиент не понимает, кому доверяет рот. Идёт к конкурентам.
   `Критично`

**Sample Fix:**
- ДО: "Добро пожаловать в стоматологию «Улыбка»"
- ПОСЛЕ: "Стоматология без боли в Москве — приём сегодня, фиксированные цены, рассрочка 0%"

### Стилизация

- Все цифры score — `JetBrains Mono`, крупно, glow effect
- Чекбоксы статуса — кастомные, не дефолтные:
  - ✗ Critical — красный квадрат с border-glow `--glow-red`
  - ⚠ Warning — амбер ромб
  - ✓ OK — зелёный круг
- Скриншот сайта — рамка с углами как на терминале/мониторе (4 уголка-маркера, не сплошная рамка)

## 10. Финальный смысл гибрида

Вся фишка продукта — **театральность диагностики**.

Пользователь приходит на тёплый, понятный лендинг. Доверяет, нажимает "Проверить".
Попадает в холодную лабораторию. Видит робота. Видит как сканируется его сайт.
Получает отчёт, написанный голосом серьёзного врача, который не сюсюкает.
Возвращается в тёплый CTA: «Хотите чтобы мы это починили?»

Если этот контраст работает — продукт работает.
Если контраст не работает (всё одинаково светлое или всё одинаково тёмное) — продукт превращается в один из десяти AI-аудиторов.

Это **главная задача дизайн-реализации**, и она важнее любых частных решений.
